# UI デザイン・ビジュアルガイドライン
*最終更新日: 2026-03-31*

このアプリケーションにおける色使いの統一ルールです。主に `Badge` と `Button` の共通部品で管理されています。

## カラー定義とセマンティクス (Badge)

バッジはデータの「状態」や「区分」を視覚的に伝えるために使用します。

```erb
<%# バッジの例 %>
<%= render "shared/badge", label: "完成品", color: :emerald %>
```

| 色 | テーマ | 製造・物流現場・システム全体での主な用途 | ER図との対応例 |
| :--- | :--- | :--- | :--- |
| **Blue** | 予定・未着手 | 作業前(未着手)、計画中、新規受付、情報ラベル | 作業指示データ(未着手) |
| **Green** | 進行・正常 | 現職のユーザー、順調に進行している作業、稼働中、納期に余裕あり | 作業実績.総合合否(合格) |
| **Yellow** | 注意・遅延 | 異常はあるが進行している作業、手直し中、要注意、納期間近 | 作業実績.不合格区分(手直し) |
| **Red** | 異常・停止 | 異常による停止、スクラップ(廃棄)、エラー、削除済、納期遅延 | 作業実績.不合格区分(スクラップ) |
| **Gray** | 完了・無効 | 問題なく完了・出荷済みの作業、退職したユーザー | discarded_at がある状態 |
| **Purple** | 特殊・管理 | 管理者設定、システムマスタの操作など(日常業務と区別) | org_unit_permissions |

## アクションの種類 (Button)

ボタンは「何ができるか」の予測可能性を高めるために、色を統一します。

```erb
<%# ボタンの例 %>
<%= render "shared/button", label: "出荷を確定する", path: ship_path, color: :emerald %>
```

| 色 | 役割 | 具体的なアクション例 |
| :--- | :--- | :--- |
| **Blue** | 決定・実行（Primary） | 保存、新規作成、検索実行、承認、出荷確定 |
| **Green** | 編集・変更（Edit） | 編集画面へ移動、情報の修正を開始 |
| **White** | 取消・戻る（Secondary） | 戻る、キャンセル、詳細を閉じる |
| **Red** | 危険・破壊（Danger） | 削除、工程の致命的な停止 |
| **Yellow** | 注意・中断（Warning） | 手直し指示、差戻し、保留 |
