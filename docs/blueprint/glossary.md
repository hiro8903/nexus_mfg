# 用語集 (Glossary)

プロジェクト全体で使用する用語の定義です。
ドキュメント、コード、UI上の表記にブレが生じないよう、ここに定義された用語を正として扱います。

---

## 組織・人事 (HR & Organization)

| 日本語 | 英語 (モデル名) | 定義 | 備考 |
| :--- | :--- | :--- | :--- |
| ユーザー | `User` | システムにログインする個人 | 社員・パート等の雇用形態は問わない |
| 拠点 | `Site` | 自社の本社・工場・支店などの物理的な事業所 | 旧 Facility。人事上の「勤務地」として使用 |
| 組織単位 | `OrgUnit` | 部・課・班・PJ・委員会などの組織の最小単位 | 自己参照（parent_id）で階層を表現 |
| 配属 | `Assignment` | ユーザーと組織・拠点の紐付け（履歴管理） | 開始日・終了日で期間を管理 |

## 取引先・納入先 (Business Partners & Logistics)

| 日本語 | 英語 (モデル名) | 定義 | 備考 |
| :--- | :--- | :--- | :--- |
| 取引先 | `BusinessPartner` | 財務的な取引主体（会社・法人単位） | 顧客・仕入先・メーカーの役割をフラグで管理 |
| 顧客 | — (フラグ: `is_customer`) | 当社に受注（加工依頼）をくれる会社 | 取引先の役割の一つ |
| 仕入先 | — (フラグ: `is_supplier`) | 当社が発注（購入）する相手の会社 | 取引先の役割の一つ |
| メーカー | — (フラグ: `is_manufacturer`) | 品番（原材料）の製造元 | 仕入先と兼ねることが多い |
| 納入先 | `DeliveryDestination` | 物理的な配送先（工場・倉庫・物流拠点等） | 取引先に1対多で紐づく |

## 品目構成 (Items & BOM)

| 日本語 | 英語 (モデル名) | 定義 | 備考 |
| :--- | :--- | :--- | :--- |
| 品目マスタ | `Item` | 当社で扱う全てのモノ（受託品、購入品、内製調合品）の総称 | 役割は `item_type` 区分で判別（画面上は商品/品番等で絞り込み表示） |
| 品目構成 (BOM) | `ItemBOM` | 親品目を作るために必要な子品目のレシピ・部品表 | 同じItem同士を繋ぐ自己参照による無限階層 |

## 在庫・物流 (Inventory & Logistics)

| 日本語 | 英語 (モデル名) | 定義 | 備考 |
| :--- | :--- | :--- | :--- |
| ロケーション | `Location` | 在庫の物理的な保管場所（棚・ラック等） | 拠点（Facility）に紐づく |
| 在庫 | `Inventory` | 特定の場所にある品目の数量 | 組織単位（論理所有）とロケーション（物理場所）の両方を持つ |

## 調達 (Procurement)

| 日本語 | 英語 (モデル名) | 定義 | 備考 |
| :--- | :--- | :--- | :--- |
| 仕入見積 | `PurchaseQuote` (予定) | 仕入先から提示された単価・条件 | 発注時に参照される |
| 発注 | `PurchaseOrder` (予定) | 仕入先への購入注文 | 仕入見積に基づく |
| 入荷 | `Receipt` (予定) | 発注した資材の受け入れ処理 | 在庫の増加に直結する |

## 受注・販売 (Sales)

| 日本語 | 英語 (モデル名) | 定義 | 備考 |
| :--- | :--- | :--- | :--- |
| 受注 | `SalesOrder` (予定) | 顧客からの加工依頼・注文 | 作業指示の発生元 |
| 販売見積 | `SalesQuote` (予定) | 顧客への見積提示 | 将来実装予定 |

## UI・デザイン (Design System)

| 用語 | 定義 | 参照先 |
| :--- | :--- | :--- |
| デザインシステム | ボタン・バッジの色と役割の定義 | [`design_system.md`](./design/design_system.md) |
| 決定・実行 (Blue) | メインアクションボタンの色 | 保存、登録、確定 等 |
| 編集・変更 (Green) | 編集画面への遷移ボタンの色 | 編集、変更 等 |
| 取消・戻る (White) | キャンセルや戻るボタンの色 | キャンセル、一覧へ戻る 等 |
| 危険・削除 (Red) | 破壊的操作のボタンの色 | 削除、退職処理 等 |

---

> **運用ルール**: 新しいモデルや業務用語が登場した際は、実装前にこの用語集へ追記してから開発に着手する。
